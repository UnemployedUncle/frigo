# 요리 여정

요리 여정(Parcours Cuisine)은 냉장고에 이미 있는 재료를 기준으로 식사를 계획하고, 장보기를 정리하고, 실제 요리까지 이어지게 돕는 서비스입니다.

브랜드 내 주요 화면 이름은 다음과 같이 사용합니다.

- `Parcours Cuisine`: 홈
- `Frigo`: 냉장고
- `Plan Repas & Courses`: 레시피 추천 및 장보기
- `Etapes`: 쿠킹 모드

---

## 환경 변수 설정

이 프로젝트는 OpenRouter를 사용할 수 있도록 환경 변수 파일을 기준으로 API 키를 관리합니다.

1. 루트 디렉터리에서 `.env.example`을 참고해 `.env` 파일을 준비합니다.
2. `.env` 파일의 `OPENROUTER_API_KEY`에 본인의 OpenRouter API 키를 입력합니다.
3. 필요하면 `OPENROUTER_BASE_URL`은 기본값 `https://openrouter.ai/api/v1`을 사용합니다.

예시:

```bash
cp .env.example .env
```

`.env` 파일 예시:

```env
OPENROUTER_API_KEY=your_openrouter_api_key_here
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
```

보안을 위해 `.env`는 `.gitignore`에 포함되어 있으며, 실제 API 키는 저장소에 커밋하지 않습니다.

## 프로젝트 소개

이 서비스는 냉장고 속 재료와 유통기한을 중심으로 식사 계획을 세우도록 돕습니다. 이미 가진 재료를 먼저 활용하게 하고, 필요한 장보기만 제안하며, 요리 기록까지 한 흐름으로 연결하는 것이 목표입니다.

## 해결하려는 문제

- 냉장고 재료를 제때 쓰지 못해 버리게 되는 문제
- 매번 무엇을 먹을지 고민하느라 요리 계획이 끊기는 문제
- 필요한 것보다 더 많이 장보게 되는 문제
- 요리를 꾸준한 습관과 취미로 만들기 어려운 문제

## 핵심 가치

- 재료 기반 추천: 새로 사기 전에 이미 가진 재료부터 활용하도록 돕습니다.
- 유통기한 중심 의사결정: 빨리 소비해야 하는 재료를 우선순위에 둡니다.
- 장보기 최소화: 부족한 재료만 추려 효율적으로 구매하게 합니다.
- 요리 습관화: 계획, 장보기, 조리, 기록이 하나의 흐름으로 이어지게 설계합니다.

## 주요 기능

### 1. 홈

화면 목적: 사용자의 요리 활동을 한눈에 보여주고, 다음 요리를 시작할 동기를 제공합니다.

핵심 사용자 행동:

- 최근 요리 기록을 확인합니다.
- 완료한 레시피를 다시 찾아봅니다.
- 사진 기반 추천 레시피를 탐색합니다.

대표 기능:

- GitHub 잔디와 유사한 방식의 요리 기록 시각화
- 완료한 레시피 갤러리
- 홈에서 바로 이어지는 추천 레시피 노출

기술 메모:

- 요리 완료 기록 누적 방식 설계
- 사용자 이력 기반 추천 레시피 연결

### 2. 냉장고 재료 관리

화면 목적: 냉장고 속 재료 상태를 빠르게 입력하고, 남은 기간과 수량을 직관적으로 파악하게 합니다.

핵심 사용자 행동:

- 재료를 말이나 텍스트로 등록합니다.
- 수량과 유통기한을 수정합니다.
- 정렬 기준에 따라 필요한 재료를 우선 확인합니다.

대표 기능:

- 재료명, 수량, 유통기한, 남은 일수 표시
- 사용자 선호 기반 정렬
- 손으로 쉽게 수정 가능한 입력 UI

기술 메모:

- 자연어 입력을 구조화된 재료 데이터로 정리
- 빠른 수정에 적합한 인터랙션 설계

### 3. 레시피 추천 및 장보기

화면 목적: 빨리 사용해야 하는 재료를 우선 활용하는 레시피를 추천하고, 필요한 장보기를 함께 정리합니다.

핵심 사용자 행동:

- 현재 재고를 기반으로 추천 레시피를 확인합니다.
- 부족한 재료만 장보기 목록으로 확인합니다.
- 단일 식사 추천에서 주간 식단 계획으로 확장합니다.

대표 기능:

- 임박 재료 우선 레시피 추천
- 일간 장보기 목록 제공
- 주간 식단 계획
- 주간 장보기 최적화

예시:

- 시금치 1일 남음
- 새우 2일 남음
- 버터 10일 남음

이 경우 추천 결과는 시금치 파스타, 갈릭 쉬림프, 새우 리조또처럼 임박 재료를 먼저 활용하는 방향으로 구성됩니다.

주간 식단 계획에서는 다음 기준을 반영합니다.

- 재료 재사용 극대화
- 유통기한 우선 반영
- 추가 장보기 최소화

예시 식단:

- 월: 시금치 파스타
- 화: 새우 리조또
- 수: 데리야키 치킨
- 목: 버섯 리조또
- 금: 스테이크 샐러드

주간 장보기 목록은 다음 방식으로 정리합니다.

- 중복 재료는 합쳐서 표시
- 꼭 필요한 재료만 추출
- 냉장고 재고 사용량과 연결

기술 메모:

- 레시피 제안 로직
- 부족 재료 기반 장보기 목록 도출
- 예상 가격 정보 확장 가능성

### 4. 쿠킹 모드

화면 목적: 레시피 전체 흐름을 먼저 보여주고, 실제 요리 중에는 단계별 진행에 집중하게 합니다.

핵심 사용자 행동:

- 전체 조리 흐름을 확인합니다.
- 단계별 화면으로 요리를 진행합니다.
- 실제 소요 시간과 변경한 내용을 기록합니다.

대표 기능:

- 전체 워크플로우 조회
- 조리 시간, 재료 목록, 필요한 도구, 단계별 순서 표시
- 좌우 스와이프로 이전 단계와 완료 처리
- 타이머 지원
- 나만의 레시피로 수정 및 저장

기술 메모:

- 실제 소요 시간 기록
- 변경한 재료와 조리 흐름 반영

## MVP 범위

MVP는 아래 네 가지 흐름을 중심으로 구성합니다.

- 홈에서 요리 기록과 추천 레시피를 확인할 수 있어야 합니다.
- 냉장고 화면에서 재료, 유통기한, 정렬 정보를 관리할 수 있어야 합니다.
- 추천 화면에서 레시피와 장보기 목록을 확인할 수 있어야 합니다.
- 쿠킹 모드에서 전체 워크플로우와 단계별 조리를 수행할 수 있어야 합니다.

현재 MVP에서 우선 구현하는 요소는 다음과 같습니다.

- 요리 기록 시각화
- 냉장고 재료 관리
- 단일 레시피 추천과 장보기 목록
- 단계별 쿠킹 모드

이후 확장 가능한 요소는 다음과 같습니다.

- 주간 식단 추천
- 주간 장보기 최적화
- 사용자 맞춤형 추천 고도화

## 기술 방향

- 사진 위주의 시각적 경험을 중심으로 설계합니다.
- 개인화 에이전트 개발을 위한 피드백 루프를 구성합니다.
- 장보기 스타일, 요리 다양성, 레시피 변경 이력을 사용자 스킬로 저장하고 공유할 수 있게 확장합니다.
